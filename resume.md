### Hermes Hub｜多 Agent / 多 Profile / 多节点统一管理平台

**项目定位：** 面向 Hermes Agent 的中心化控制台，通过 Controller-Agent 架构统一管理本地和远程机器上的多个 Agent 实例。

**我关注 / 负责的产品设计点：**

- 设计 Agent Fleet 总览能力，统一展示不同节点上的 Agent，并支持按节点过滤。
- 规划 Profile 生命周期管理，包括创建、重命名、删除、配置编辑、SOUL.md 编辑、Env 管理和 Skills 管理。
- 设计节点管理流程：Web 创建节点、生成 vkey、复制部署命令、Hub Agent 注册与心跳上报。
- 设计运维控制能力：Gateway 启停、Setup / Doctor、命令队列、日志聚合和 WebSocket 状态推送。
- 引入平台产品常见能力：JWT 登录、RBAC 角色权限、审计日志、敏感信息脱敏、SQLite 持久化。

**项目价值：**

- 展示了 AI Agent 从“单体工具”走向“多实例、多节点、可运维平台”的产品化能力。
- 体现了 AI 产品经理在管理后台、权限模型、日志审计、配置系统和运维可观测性方面的设计能力。

**项目地址：** <https://github.com/JiChaoSong/hermes-hub>

## 我的 AI 产品判断

我认为 AI 产品经理的核心价值，不是把大模型接进页面，而是回答清楚以下问题：

1. 用户真正希望 AI 帮他完成什么任务？
2. 哪些环节适合自动化，哪些环节必须保留人工判断？
3. 模型输出如何评估，失败后如何纠偏？
4. 工具调用、权限、数据和成本如何被产品化管理？
5. 产品如何从 Demo 走向可用、可控、可维护、可商业化？

因此，我希望继续在 AI Agent、AI 工具产品、RAG 知识库、AI 平台产品和 AI 产品方法论方向持续深入。
